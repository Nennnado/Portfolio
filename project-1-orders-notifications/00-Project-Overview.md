# Проект 1: Система управления заявками на ТОиР оборудования

**Роль:** Системный и бизнес-аналитик (полный цикл: от интервью с пользователями до постановки задач разработчикам)

**Контекст:** Внутренняя система для компании, предоставляющей услуги технического обслуживания и ремонта промышленного оборудования (станки, компрессоры, КИПиА).

**Проблема (AS-IS):**
*   Заявки на ТОиР поступают по email, телефону и в чаты — данные теряются, нет единого реестра.
*   Статус заявки отслеживается через звонки и переписку в мессенджерах.
*   Непонятно, кто из инженеров свободен, а кто занят. Назначения происходят вручную "на глазок".
*   Невозможно построить отчет по выполненным работам, загруженности специалистов и SLA.

**Цель проекта (TO-BE):**
Разработать централизованную систему учета заявок на ТОиР, которая позволит:
1.  **Снизить время обработки заявки** с момента поступления до назначения инженера на 30%.
2.  **Повысить прозрачность** для менеджеров и клиентов за счет отслеживания статуса в реальном времени.
3.  **Оптимизировать загрузку инженеров** за счет автоматического учета их специализации и текущей занятости.
4.  **Автоматизировать формирование отчетов** по выполненным работам.

**Ключевые возможности (Scope):**
*   Создание, просмотр, редактирование и фильтрация заявок.
*   Назначение/переназначение инженеров на заявки с учетом их специализации.
*   Ведение жизненного цикла заявки (статусы: `Новая` → `В работе` → `На паузе` → `Выполнена` → `Закрыта`).
*   Просмотр личного кабинета инженера с его задачами.
*   Базовые отчеты для руководителя отдела.
