# Статусная схема заказов и операций

## Суть проекта
На проекте существующая система статусов работает нелогично, из-за чего регулярно возникают баги и ошибки в бизнес-процессах.

## Цель
Проанализировать текущую статусную схему операций и заказов (**AS IS**) и спроектировать корректную статусную схему (**TO BE**), которая:

- соответствует логике бизнес-процессов;
- правильно отображает текущее состояние заказа и операций;
- минимизирует ошибки и баги.

## Задачи проекта
1. Провести анализ текущей схемы статусов (**AS IS**) для операций и заказов.
2. Спроектировать целевую схему статусов (**TO BE**) с учётом корректной логики и возможных переходов.
3. Разработать методы для переключения статусов операций и заказов (API, кнопки, условия).
4. Передать проект на бэкенд для реализации и интеграции.

## Результат
- Чёткая и непротиворечивая схема статусов операций и заказов.
- Упрощённая логика работы приложения для пользователей.
- Минимизация багов, связанных с некорректными статусами.
- Документированные методы и правила переходов статусов для бэкенда.
